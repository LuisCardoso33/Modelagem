Sistema de Cardápio de Lanches

Data de geração: 11/06/2025

1. Visão de Raio-X

Entrada: Seleção de lanche pelo usuário via menu suspenso.
Processamento: JS lê o objeto correspondente, renderiza ingredientes e preço.
Saída: Lista de ingredientes, preço individual e pedido acumulado.
Estado do Sistema: Armazenado em duas variáveis principais:
- pedido[]: array com todos os itens escolhidos
- total: soma acumulada dos preços
Componentes principais:
  • HTML: Interface com select, listas e botão
  • JavaScript: Lógica de exibição e controle de estado
  • CSS: Aparência e responsividade

2. Descrição Geral

Este sistema é um cardápio digital interativo que permite ao usuário selecionar lanches, visualizar seus ingredientes e preços, adicionar ao pedido e visualizar o total acumulado. Ideal para lanchonetes, sistemas de autoatendimento ou projetos didáticos.

3. Tecnologias Utilizadas

  • HTML5: Estrutura da página
  • CSS3: Estilização da interface
  • JavaScript: Lógica e interatividade do sistema

4. Funcionalidades

  • Lista de lanches com nomes, ingredientes e preços
  • Exibição dinâmica dos ingredientes ao selecionar um lanche
  • Botão para adicionar lanche ao pedido
  • Atualização automática da lista de pedidos e cálculo do total

5. Estrutura do Código

HTML: Define os elementos visuais da aplicação (select, lista de ingredientes, botão de adicionar e lista de pedidos).
CSS: Define estilo da página, como cores, espaçamento, bordas e tipografia.
JavaScript:

  • Objeto 'lanches': contém os dados dos produtos
  • Eventos: 'change' no select e 'click' no botão de adicionar
  • Funções: atualização da lista de ingredientes, pedidos e cálculo do total

6. Melhorias Sugeridas

  • Permitir remoção de itens do pedido
  • Implementar quantidades para cada item
  • Adicionar imagens ilustrativas dos lanches
  • Salvar pedidos usando localStorage ou banco de dados
  • Gerar recibo em PDF para impressão

7. Considerações Finais

O projeto demonstra habilidades básicas em desenvolvimento web front-end, manipulação de DOM e controle de estado do pedido. É uma excelente base para expandir e transformar em um sistema mais robusto com backend.
